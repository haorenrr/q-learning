以当前对量子计算优势的理解，较难、也没有必要完全取代传统计算机，因此目前主流的是量子计算机+经典计算机的「超量」架构。如上图，虚线框外由经典计算机完成，主要包括控制、数据初始化、结果处理等工作；虚线框内则由量子计算机执行，主要包括在量子叠加态上高效执行量子算法、测量与结果回传等工作。

量子程序的执行过程，分为三个步骤：初态制备、幺正变换和对末态的测量。即总体看，量子程序就是完成变换 $\ket {\psi_{final}} = U\,\ket {\psi_0}$ ，其中：
1. $\ket {\psi_0}$ 代表量子线路的初态，需要根据业务要求，把它设置为恰当的初始值。这一步是经典世界通往量子世界的传送门。
2. 上式中的 $U$ ，就是设计的量子算法。对于复杂算法，U可以拆解为一组子变换的乘积，即 $U=U_1U_2\cdots U_n$ 。这一步完全工作在量子世界。
3. $\ket {\psi_{final}}$ 代表可测量的末态。这一步需要通过测量，让量子态坍塌到某个本征态，再把结果传回到经典计算机。这一步是量子世界回到经典世界的传送门。

关于末态的测量结果，本质上是概率性的，由各本征态的概率幅决定，因此一个量子算法需要重复测量多次，才能获得统计学意义上的置信度。

既然神奇的魔法（算法）发生在量子世界，但我们又是活在经典世界的，那么连接量子世界和经典世界的传送门就非常关键了，这分别是初态制备和末态测量要做的事情。

### 初态制备

算法开始前，量子比特会被初始化为本征态$\ket 0$ ，这是底层硬件提供的能力，因此一切都是从一个确定的状态开始的。
#### 示例一
某算符需要以$\ket{q_0q_1q_2} = \ket{110}$ 作为输入，即在三量子比特系统中，初始化为某个本征态，不存在态的叠加，这种情况最简单，只需要使用$X$ 门把对应位反转即可，线路图如下：

#### 示例二

假设某算符$U$， 需要以状态$\ket \psi$ 作为输入，其中 $\ket \psi = \frac{1}{\sqrt 2}(\ket 0 + \ket 1)$ ，此时态矢位于布洛赫球上x轴正向，只需要对$\ket 0$ 施加H门：

#### 示例三
假设某算符$U$，需要以状态$\ket \psi$ 作为输入，其中$\ket \psi = \frac{1}{2}(\ket {00} + \ket {01} + \ket {10} + \ket {11})$ ，这是个2 qubit系统，分析发现表达式可做因式分解：
$$
\begin{align*}
\ket \psi &= \frac{1}{2}(\ket {00} + \ket {01} + \ket {10} + \ket {11}) \\
	&=\frac{1}{\sqrt 2}(\ket 0 + \ket 1)\, \otimes \,\frac{1}{\sqrt 2}(\ket 0 + \ket 1) \\
	&= \ket {q_0} \otimes \ket {q_1} 
\end{align*}
$$
状态可以分解为两个独立量子比特的张量积，说明$q_0,q_1$ 不是纠缠态，分别对他们进行幺正变换即可，线路图如下

![300](attachments/Pasted%20image%2020260410110304.png)

图中虚线框代表的算符，可简写为$H^{\otimes 2}$，$H^{\otimes n}$ 表示$H$分别作用在$n$ 个量子比特上。

对于元素相等的数组，可以采用这种编码方式，比如$X=[1,1,1,1]$ ，即把元素中的数字做归一化之后，编码到各本征态的系数中，这种编码方式叫做振幅编码。这类各元素都相等的数组编码后的量子态，叫均匀叠加态，但这种情况毕竟是少数，对于任一的数组，比如$X=[1,2,3,4]$ 该如何处理呢？继续看下个例子
#### 示例四

数组$X=[1,2,3,4]$ 要编码到量子态中，需要先归一化，即$x_1^2 + x_2^2 + x_3^2 + x_4^2 = 1$ ，处理后的数据，采用振幅编码，就是

$$
\begin{align*}
&\text{由} \quad 1^2 + 2^2 + 3^2 + 4^2 = 30 \\
&\text{得} \quad \ket \psi = \frac{1}{\sqrt {30}}(\ket {00} + 2\ket {01} + 3 \ket {10} + 4 \ket {11}) 
\end{align*}
$$
但是这个表达式无法做因式分解，即无法表示成独立量子比特的张量积，这是一个纠缠态，不能像上例那样，分别对两个量子比特操作来实现了。

可做如下分析， 上式采用的是形如 $\ket {q_0q_1}$ 的表示，即高位为$q_0$ ：
1. $q_0$为0的概率是 $P(q_0 = 0) = \frac{1}{30}(1^2 + 2^2) = \frac{5}{30}$ 
2. 当$q_0 = 0$ 时，低位$q_1 = 0$ 的概率是 $\frac{2^2}{1^2 + 2^2} = \frac{4}{5}$ ，$q_1 = 1$ 的概率是 $1 - \frac{4}{5} = \frac{1}{5}$ 
3. 当$q_0 = 1$ 时，低位$q_1 = 0$ 的概率是 $\frac{4^2}{3^2 + 4^2} = \frac{16}{25}$ ，$q_1 = 1$ 的概率是 $1 - \frac{16}{25} = \frac{9}{25}$ 

大思路是，先处理 $q_0$ 的概率，然后在 $q_0$ 的不同分支处理$q_1$ ，采用的是二分法从顶往下拆解的递归思路。这种方式也叫做 Top-Down 振幅编码。

按照Top-Down 振幅编码的思路，线路图可以这么设计：
1. 先让量子比特$q_0$ 绕$y$ 轴逆时针旋转$\theta_{q0} = 2\arccos(\sqrt{\frac{5}{30}})$ ，即使用算符$R_y(\theta_{q0})$ ，这一步可以确定$q_0$ 的概率分布
2. 当$q_0 = 0$ 时，量子比特$q_1$ 绕y轴逆时针旋转$\theta_{q1} = 2\arccos(\sqrt{\frac{4}{5}})$，即使用算符$R_y(\theta_{q1})$
3.  当$q_0 = 1$时，量子比特$q_1$ 绕y轴逆时针旋转$\gamma_{q1} = 2\arccos(\sqrt{\frac{16}{25}})$，即使用算符$R_y(\gamma_{q1})$

量子线路图如下：

![300](attachments/Pasted%20image%2020260410130008.png)

采用振幅编码的方式，可以对任意长度的数据进行初始化，且N个数据点时，只需要$log_2N$ 个量子比特，比如存储10亿个数据点，仅需30 qubit，这种编码方式下，量子比特的信息密度是远高于经典比特的。

但振幅编码也存在一个重要问题，就是初始化线路太深，可以证明有N个数据点时，把这些数据初始化进量子比特中，需要N个$Ry$ 门，这会带来两个问题：1）量子比特的有效时间很短，若需要执行的门数太多，数据还未初始化完，系统就因退相干而失效了，2）不少量子算法声称可以做到指数级加速 ，但加上数据初始化耗时的话，这个加速优势很容易就被抵消了。

数据初始化作为进入量子世界的传送门，这一步的效率，决定了整个系统的成败，因此数据初始化是目前一个比较重要的研究方向。

### 末态测量

量子测量的本质是概率性的。 即使你制备了一个完美的态 $|\psi\rangle = a|0\rangle + b|1\rangle$，单次测量也只能得到一个 $0$ 或 $1$。为了逼近理论上的概率 $|a|^2$ 和 $|b|^2$，则必须进行大量重复实验。

要命的是，测量会立即引起量子态坍塌，这就为测量带来了额外的难度。经典物理中，通过多次测量消除误差的优势，在这里变成了劣势，因此必须打破常规引入新思路。

简言之，要确定一个量子态，必须满足如下条件：
1. 多次测量，通过$\ket 0, \ket 1$ 的概率分布，还原量子态
2. 由于量子态无法直接克隆（备份），为了支持多次测量，系统必须可重复制备，多次运行相同算法，得到相同末态。

我们知道，传统程序只需执行一次就可得到结果，量子算法则很不一样，为了得到置信的结果，它必须重复「制备-执行-测量」这个循环很多次。

测量次数与量子比特数、和末态本身都有密切关系。一般来说：
* 情况一：量子算法会把末态变换为某个本征态、或接近本征态，这时每次测量大概率都能得到相同结果。这种情况，一般测量1000次左右，就能达到不错的准确率。
* 情况二：末态属于均匀叠加态，比如 “$\ket \psi = \frac{1}{2}(\ket {00} + \ket {01} + \ket {10} + \ket {11})$”，而我们又打算得到每个本征态的振幅，这是最「差」的情况。此时测量次数随量子比特$n$呈指数增长，$n$ 较小时，还可以应付，但$n$较大时，就是一种灾难，比如20，就需要测量上千万次，量子计算的优势被消耗殆尽。
* 情况三：测量是算法中的一环，比如HHL算法中，需要测量辅助比特，若测得结果$\ket 1$，则在其他未坍塌的量子比特上继续工作，否则马上从零重算。这种情况下，测量作为算法的一部分而存在，测量次数虽不确定，但只要得到预期中的状态就停止测量。

因此好的算法，会把末态变换为某个（或少数几个）本征态，把复杂的处理留在量子世界内完成，确保结果可以高效的传递给经典世界。

### 小结

本节探讨了进入和跳出量子世界的两个传送门，初态制备和末态测量。可以发现它们有共同的特点，就是既关键、又都面临根本性挑战！因此可以一句话概括量子计算的特点：量子世界是无限潜力的魔法盒，但通往量子世界的传送门则没那么宽阔！
